\# Results



This directory stores CSV output files generated by the navigation algorithm.

Large result files are excluded from Git by default (see `.gitignore`).



\---



\## Output File Format



Each run of the navigator produces one CSV file.

Each row corresponds to one navigation step (default step size: 0.30 m).



\### Column Descriptions



| Column | Unit | Description |

|---|---|---|

| `step\_id` | — | Sequential step index starting at 0 |

| `centroid\_x` | m | X coordinate of the robot triangular footprint centroid |

| `centroid\_y` | m | Y coordinate of the robot triangular footprint centroid |

| `centroid\_z` | m | Z coordinate (elevation) conformed to navigable surface |

| `step\_length` | m | Euclidean distance from previous centroid to current centroid |

| `open\_space\_width\_left` | m | Fan-based lateral navigable clearance to the left |

| `open\_space\_width\_right` | m | Fan-based lateral navigable clearance to the right |

| `surface\_normal\_x/y/z` | — | Unit normal vector of the local surface plane under the robot |

| `slope\_angle` | degrees | Running slope: deviation of surface normal from vertical (0–90°) |

| `cross\_slope\_angle` | degrees | Lateral surface tilt perpendicular to heading; positive = right side down |

| `surface\_class` | code | Semantic class at the centroid position (see data/README.md) |

| `heading\_x/y/z` | — | Robot heading unit vector |

| `distance\_to\_goal` | m | Straight-line distance from current position to goal |

| `navigation\_state` | string | Algorithm state: FOLLOW\_CENTERLINE, AVOID\_OBSTACLE, GOAL\_REACHED, etc. |

| `distance\_to\_nearest\_obstacle` | m | Minimum ray-cast distance to any non-navigable point (36 directions) |

| `path\_width` | m | Total navigable corridor width (left + right, fan-based) |

| `centerline\_offset` | m | Lateral deviation from corridor centre; positive = left of centre |

| `obstacle\_dir\_0` … `obstacle\_dir\_35` | m | Ray-cast distances in 36 uniformly sampled directions (0–360°) |



\### Alignment with MEA Framework



The output parameters align with the

Measure of Environmental Accessibility (MEA) framework (Gamache et al., 2019):



| MEA Feature | CSV Column(s) |

|---|---|

| Surface type | `surface\_class` |

| Path width | `path\_width`, `open\_space\_width\_left`, `open\_space\_width\_right` |

| Running slope | `slope\_angle` |

| Cross slope | `cross\_slope\_angle` |

| Obstacle clearance | `distance\_to\_nearest\_obstacle`, `obstacle\_dir\_\*` |



\### ADA / IBC Reference Thresholds



The algorithm records \*\*continuous raw measurements\*\*, not binary pass/fail values,

enabling personalized accessibility assessment for individual user profiles.



| Parameter | ADA/IBC threshold | Column |

|---|---|---|

| Running slope | ≤ 5% (2.86°) | `slope\_angle` |

| Cross slope | ≤ 2% (1.15°) | `cross\_slope\_angle` |

| Step height | ≤ 6.4 mm | (step detection module) |

| Path width (single passage) | ≥ 0.91 m (36 in) | `path\_width` |

| Path width (passing space) | ≥ 1.52 m (60 in) | `path\_width` |

| Ramp slope | ≤ 8.33% (4.76°) | `slope\_angle` on ramp segments |

\## Validation Accuracy (from the paper)



Quantitative comparison over a 22.5 m sidewalk segment (75 steps):



| Parameter | MAE | RMSE | MBE | NRMSE |

|---|---|---|---|---|

| Path width | 0.027 m | 0.033 m | 0.005 m | \*\*2.71%\*\* |

| Running slope | 0.776° | 0.962° | 0.062° | 66.56% |

| Cross slope | 1.315° | 1.594° | 1.315° | 64.89% |



> Source: Ahmadi et al. (2026), \*Sensors\*, Table 2.
